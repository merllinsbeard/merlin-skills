# Merlin Skills v0.2.0

Promotes GitHub Spec Kit from archive-only provenance to first-class runtime skills for Codex.

## Added

- Official GitHub Spec Kit Codex skills:
  - `speckit-constitution`
  - `speckit-specify`
  - `speckit-clarify`
  - `speckit-plan`
  - `speckit-tasks`
  - `speckit-analyze`
  - `speckit-checklist`
  - `speckit-taskstoissues`
  - `speckit-implement`
- Repo-owned `speckit-cli` skill for `specify` install, version checks, project init, integrations, extensions, and presets.

## Changed

- Refreshed `github/spec-kit` provenance to official release `v0.8.12`.
- Rewrote `merlin-skills-routing` as a phase router for the Merlin `/goal` process.
- Kept `speckit-implement` available for explicit upstream-native implementation, but preserved `create-goal -> /goal` as the default Merlin path.
- Kept `speckit-taskstoissues` explicit because it creates GitHub-side artifacts.
- Left `speckit-git-*` as CLI/extension-only instead of default installed skills.
- Required a short Spec Kit consultation before non-trivial skill decisions.
- Clarified that `tdd` is the vertical-slice implementation discipline after `tasks.md`, `GOAL.md`, and `/goal`, not a replacement for `speckit-tasks`.
- Updated docs, validation, third-party notices, and manifest for 23 installable skills.
